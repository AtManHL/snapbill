# 秒拍帐小程序PRD文档

# 1. 文档概述

## 1.1 文档目的

本文档为「秒拍帐」微信小程序开发需求说明，明确项目背景、核心功能、交互逻辑、开发约束等，为Claude Code开发提供清晰依据，确保开发成果贴合用户实际需求（解决记账繁琐、易遗忘、多人共享不便等痛点）。

## 1.2 项目基本信息

- 项目名称：秒拍帐（微信小程序）

- 开发工具：Claude Code、微信小程序开发者工具

- 核心定位：AI自动识别支付截图记账，支持多人共享账本，极简操作、高效便捷

- 目标用户：有共同记账需求的群体（如夫妻、室友等），追求简洁、高效记账，厌恶繁琐手动输入的用户

# 2. 项目背景与痛点

## 2.1 项目背景

目标用户（开发者与配偶）存在共同记账需求，但现有记账软件操作繁琐，需手动输入金额、选择分类，易遗忘记账步骤；同时，当前95%以上的支出为手机支付，具备“截图提取信息”的场景基础，因此开发一款极简、AI驱动、支持多人共享的记账小程序，解决日常记账痛点。

## 2.2 核心痛点

- 操作繁琐：传统记账需手动输入金额、选择分类，步骤多，用户易放弃、易遗忘；

- 多人共享不便：现有软件多人共享账本操作复杂，无法快速识别不同记账人身份；

- 分类纠结：手动选择消费分类时，易出现分类模糊、选择耗时的问题。

## 2.3 解决思路

基于手机支付截图场景，利用AI自动提取支付金额、支付时间，智能匹配消费分类；结合微信账号识别，实现多人共享一个账本，记账人身份自动关联；简化操作流程，用户仅需“截图→上传→确认”三步完成记账，降低记账门槛。

# 3. 核心功能需求

## 3.1 功能整体框架

核心功能分为三大模块：AI截图识别记账模块、多人共享账本模块、账本统计分析模块；辅助功能包括：个人中心、账本设置、分类管理、历史记录查询。

## 3.2 详细功能需求

### 3.2.1 AI识别支付截图记账（核心功能）

#### 3.2.1.1 截图上传

- 支持从手机相册上传支付截图（适配微信支付、支付宝支付等主流手机支付截图格式）；

- 支持拍照实时上传（用户消费后直接拍照，无需保存相册，简化操作）；

- 上传后给出“识别中”提示，识别失败则提示“识别失败，请重新上传清晰截图”。

#### 3.2.1.2 AI自动提取信息

- 核心提取字段：支付金额（自动识别数字，区分收入/支出，默认支出，可手动修改）、支付时间（提取截图中的支付日期、时间，精确到分钟）；

- 辅助提取字段（可选）：支付商户名称（如“美团外卖”“超市”），用于辅助智能分类；

- 识别精度要求：主流支付截图识别准确率≥95%，模糊、遮挡截图给出明确失败提示。

#### 3.2.1.3 智能分类与确认

- 根据提取的商户名称、金额范围，智能匹配消费分类（如“餐饮”“交通”“购物”“水电煤”等，默认提供8-10个常用分类）；

- 识别完成后，展示“金额+时间+分类”预览界面，用户可手动修改分类、金额、时间（满足特殊场景需求）；

- 用户点击“确认记账”，完成本次记账，跳转至账本首页，给出“记账成功”提示；点击“取消”，返回上传页面。

### 3.2.2 多人共享账本（核心功能）

#### 3.2.2.1 账本创建与管理

- 用户首次进入小程序，自动创建默认账本（名称可修改，如“我们的小家账本”）；

- 支持创建多个账本（如“家庭账本”“朋友聚会账本”），可设置账本名称、备注；

- 支持删除、切换账本（切换后展示对应账本的记账记录与统计数据）。

#### 3.2.2.2 多人共享邀请

- 账本创建者可生成共享邀请码/邀请链接，通过微信分享给他人；

- 被邀请者通过微信点击链接/输入邀请码，即可加入该账本（无需注册，直接通过微信账号关联）；

- 支持设置共享权限：仅创建者可删除账本、管理成员；所有成员可查看账本、添加记账记录、修改自己的记账记录。

#### 3.2.2.3 记账人自动识别

- 每个成员加入账本后，自动关联自己的微信昵称、头像，作为记账人标识；

- 成员添加记账记录时，记账人字段自动填充为当前微信账号对应的身份，无需手动选择；

- 账本中所有记录均显示记账人信息，便于区分不同成员的消费。

### 3.2.3 账本分类统计与查询（核心功能）

#### 3.2.3.1 分类统计

- 按消费分类统计：展示每个分类的支出金额、占比（以图表形式呈现，如饼图、柱状图，简洁直观）；

- 按时间统计：支持按日、周、月、年查看记账总金额、各分类金额，可切换时间维度；

- 按记账人统计：支持查看每个成员的总支出、各分类支出，便于对账（如夫妻间核对各自消费）。

#### 3.2.3.2 分类管理

- 支持自定义消费分类：可添加、删除、修改分类名称（如添加“宠物用品”“房贷”等个性化分类）；

- 支持设置常用分类：常用分类优先展示，便于手动修改时快速选择；

- 分类删除约束：若分类下有记账记录，删除时提示“该分类下有历史记录，删除后记录将归为‘其他’分类”，确认后可删除。

#### 3.2.3.3 历史记录查询

- 支持按关键词搜索：输入金额、商户名称、分类，快速查询对应记账记录；

- 支持筛选查询：按时间范围、记账人、分类筛选记录；

- 每条记录可点击查看详情（金额、时间、分类、记账人、截图缩略图），支持编辑、删除（仅可编辑/删除自己添加的记录）。

### 3.3 辅助功能需求

- 个人中心：展示微信昵称、头像，可修改个人备注（便于账本成员区分），查看我的记账统计；

- 记账提醒（可选）：支持设置每日固定时间提醒（如21:00），提醒用户上传当日支付截图记账；

- 数据备份与恢复：支持将账本数据导出为Excel（便于本地备份），支持从Excel导入数据（仅创建者可操作）；

- 错误处理：网络异常、AI识别失败、邀请码无效等场景，给出清晰、易懂的提示，引导用户操作。

# 4. 交互需求

## 4.1 整体交互原则

极简操作，减少跳转；核心流程（截图→上传→确认）不超过3步；反馈及时，操作后给出明确提示（如记账成功、识别失败）；适配微信小程序操作习惯，避免复杂手势。

## 4.2 核心流程交互

### 4.2.1 记账流程

首页点击“+记账”→ 选择“上传截图”/“拍照”→ 上传完成后自动识别，跳转至确认页面（展示金额、时间、分类）→ 用户确认/修改后点击“确认记账”→ 跳转首页，提示“记账成功”。

### 4.2.2 多人共享流程

账本详情页点击“共享账本”→ 生成邀请码/链接→ 分享给微信好友→ 好友点击链接/输入邀请码→ 自动加入账本，跳转至该账本首页，提示“已加入XX的账本”。

### 4.2.3 统计查询流程

首页点击“统计”→ 默认展示当月分类统计（饼图）→ 可切换时间维度（日/周/年）、统计类型（分类/记账人）→ 点击图表可查看对应明细；点击“查询”，可输入关键词/筛选条件查询历史记录。

# 5. 开发约束

- 开发平台：基于微信小程序开发者平台开发，适配微信小程序最新版本，兼容iOS、Android双端；

- AI识别：集成Claude Code的AI识别能力，重点实现支付截图的金额、时间、商户信息提取，以及智能分类匹配；

- 数据存储：用户数据、账本数据、记账记录、截图缩略图（原图不存储，仅存储识别后的信息）存储在微信小程序云开发数据库（便捷、低成本，适配个人开发）；

- 兼容性：适配主流手机型号（屏幕尺寸4.7-6.7英寸），兼容微信版本8.0及以上；

- 安全性：通过微信账号授权登录，无需额外注册；仅账本成员可查看对应账本数据，保护用户隐私；截图识别仅用于提取记账信息，不泄露用户其他隐私。

## P1（上线后快速迭代功能）

1. 记账提醒；2. 数据导入；3. 图表样式优化；4. 个性化账本设置（封面、颜色）。

# 8. AI功能接入说明（新增）

## 8.1 AI接入核心目标

实现支付截图（微信、支付宝为主）中关键信息（金额、时间、商户名称）的自动提取，以及消费分类智能匹配，满足识别准确率≥95%的核心要求，同时控制接入成本，适配个人开发的低成本需求，结合Claude Code开发效率优势，简化接入流程。

## 8.2 微信小程序AI接入步骤（适配现有开发框架）

结合微信小程序云开发特性及大模型API调用规范，接入流程分为4步，无需复杂部署，贴合个人开发场景，可配合Claude Code快速实现代码编写：

1. 准备工作：开通微信小程序云开发环境，获取envId并在app.js中完成云环境初始化，确保小程序基础库版本不低于2.25.0，启用增强编译以便处理现代JavaScript语法；同时注册目标大模型平台账号，获取API密钥（敏感密钥需通过云函数代理，避免前端暴露）。

2. 截图处理与上传：通过微信小程序chooseMedia API实现截图上传/拍照功能，将本地截图路径转换为Base64编码（适配大模型API要求，避免文件大小限制问题），通过云函数将Base64编码后的图片数据传递至大模型API接口。

3. 大模型调用与信息提取：在云函数中编写API调用逻辑，传入Base64图片数据及精准提示词（如“提取此支付截图的金额、支付时间、商户名称，金额区分收入/支出，时间精确到分钟，输出结构化JSON格式”），获取模型返回结果并解析，筛选出核心记账信息；若识别失败，返回“识别失败，请重新上传清晰截图”提示。

4. 结果同步与优化：将解析后的核心信息（金额、时间、分类）同步至小程序前端，展示给用户确认；同时记录识别日志，针对模糊、遮挡截图的识别失败案例，优化提示词或更换模型量化版本，提升识别精度；可借助Claude Code优化云函数调用逻辑，减少冗余代码，提升响应速度。

## 8.3 高性价比大模型选型推荐（精准+便宜，适配个人开发）

优先选择“中文识别精准、支付场景适配性强、按调用量计费、低成本”的大模型，结合支付截图识别场景（结构化信息提取，无需复杂视觉分析），推荐3款主流模型，对比后可按需选择，均支持API调用且适配微信小程序云开发架构：

### 8.3.1 首选：阿里通义千问 Qwen2.5-VL（性价比之王）

- 核心优势：中文识别精度高，对微信/支付宝支付截图的适配性强，能精准提取金额、时间等结构化信息，同时支持商户名称识别辅助分类；支持Base64图片输入，API调用稳定，通过阿里云DashScope平台提供服务，无需自行部署服务器；新用户有免费调用额度，后续按调用量计费，单价低（1000次调用约1-2元），完全适配个人开发的低成本需求。

- 适配性：支持微信小程序云函数调用，可通过简单的HTTP请求实现交互，截图识别准确率可达96%-98%，满足项目核心精度要求；对模糊截图有一定容错性，可通过优化提示词进一步提升识别效果。

### 8.3.2 备选1：智谱 GLM-4v-9b（精度突出，部署灵活）

- 核心优势：视觉理解能力强，支持高清分辨率输入，能清晰识别截图中的小字、模糊数字，识别准确率可达97%-99%，尤其适合复杂场景（如折叠截图、部分遮挡截图）；支持INT4量化版本，部署成本低，单张普通显卡即可本地部署（适合长期大量使用，降低调用成本），也可直接调用API按次计费，单价略高于Qwen2.5-VL但精度更优。

- 适配性：支持API调用和本地部署两种方式，API调用可直接对接微信小程序云函数；中文优化出色，能精准理解支付截图的结构化信息，同时可辅助实现消费智能分类，减少人工确认成本。

### 8.3.3 备选2：腾讯云 Hunyuan-VL（适配微信生态，部署便捷）

- 核心优势：与微信小程序、腾讯云开发无缝衔接，调用延迟低（无需跨平台交互），稳定性强；对微信支付截图的识别做了专项优化，金额、时间提取准确率高，支持智能分类；新用户免费额度充足，按调用量计费，单价适中（1000次调用约2-3元），适合偏好腾讯生态、追求部署便捷性的开发场景。

- 适配性：可通过微信小程序云开发AI+能力直接调用，无需额外配置跨域，代码编写更简洁，可配合Claude Code快速完成调用逻辑开发，降低开发难度。

## 8.5 微信小程序接入腾讯云Hunyuan-VL 详细步骤（专属补充）

结合本项目“支付截图识别”核心需求，适配微信小程序云开发架构，无需复杂部署，步骤清晰可落地，配合Claude Code可快速生成对应代码，全程贴合个人开发场景，同时依托腾讯生态优势，减少跨平台交互问题。

### 8.5.1 前置准备（必做）

1. 开通微信小程序云开发环境：进入微信开发者工具，打开本小程序项目，开通云开发（未使用过云开发的用户首月免费），获取云环境ID（envId），确保小程序基础库版本不低于3.7.1（支持wx.cloud.extend.AI接口），启用增强编译功能以兼容现代JavaScript语法。

2. 开通腾讯云Hunyuan-VL服务：登录腾讯云官网，搜索“混元大模型 Hunyuan-VL”，开通对应服务，领取新用户免费额度（自动领取安心体验模式50万tokens，满足初期开发测试需求），获取API Key（敏感信息，严禁前端暴露）。

3. 环境配置：在小程序app.js中完成云环境初始化，同时在云开发控制台中，配置Hunyuan-VL的API Key（通过云函数代理调用，避免前端泄露密钥），确保云函数拥有网络调用权限。

### 8.5.2 核心接入步骤（4步完成，适配截图识别场景）

1. 步骤1：截图上传与格式处理（复用项目现有截图上传逻辑）
                  通过微信小程序chooseMedia API，实现“相册上传截图”“实时拍照”功能（与3.2.1.1截图上传功能完全兼容）；

2. 将上传的截图本地路径，转换为Base64编码（适配Hunyuan-VL API要求，同时压缩图片体积，保留金额、时间等核心信息，减少调用耗时和费用）；

3. 通过wx.cloud.callFunction方法，将Base64编码后的图片数据，传递至云函数（避免前端直接调用API，保障密钥安全）。

4. 步骤2：创建云函数（核心，调用Hunyuan-VL API）
                  在微信开发者工具中，新建云函数（命名为hunyuanImageRecognition），引入wx-server-sdk，配置云环境ID；

5. 在云函数中编写调用逻辑，引入Hunyuan-VL API相关依赖（无需额外安装SDK，通过wx.cloud.extend.AI接口直接调用，适配微信云开发AI+能力）；

6. 传入核心参数：Base64图片数据、精准提示词（贴合本项目需求，示例：“提取此支付截图的金额、支付时间、商户名称，金额需区分收入/支出，时间精确到分钟，消费分类匹配餐饮、交通、购物等常用分类，输出结构化JSON格式，无需多余描述”）；

7. 配置API Key：通过云函数环境变量配置Hunyuan-VL的API Key，代码中通过process.env获取，避免硬编码泄露。
